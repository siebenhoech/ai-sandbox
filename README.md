# AI Coding Sandbox

An isolated, disposable virtual machine (VM) environment designed for safe AI-assisted programming (Agentic Coding). It features a persistent workspace, Java/Spring Boot readiness, and nested virtualization for Podman containers.

## Prerequisites
* Host OS: Linux Mint (or other Linux distributions)
* Hypervisor: VirtualBox
* Provisioner: Vagrant (`sudo apt install vagrant`)

---

## 🚀 Initial Setup (Run Once)

Before starting the VM for the first time, you need to set up the local directory structure and generate the SSH key that the AI agent will use to communicate with GitHub.

### 1. Create Directories and Generate the Git SSH Key
Run the following commands on your host machine:

```bash
# Create the secrets and workspace directories
mkdir -p ~/ai-sandbox/secrets
mkdir -p ~/ai-sandbox/workspace

# Generate a persistent SSH key for the VM to use with Git (No passphrase)
ssh-keygen -t ed25519 -C "ai-agent-key" -f ~/ai-sandbox/secrets/id_ed25519 -N ""
```

### 2. Boot the VM
Navigate to the sandbox folder and start the VM:

```bash
cd ~/ai-sandbox/vagrant
vagrant up
```

### 3. Configure SSH Access
To be able to connect to the VM smoothly, you need to add the Vagrant connection details to your host's SSH config.

Run this command to see the dynamic SSH configuration generated by Vagrant:
```bash
vagrant ssh-config
```

Copy the relevant details and add them to your host's ~/.ssh/config file. It should look like this:
Important: change the "Host default" to "Host ai-sandbox" or what you like.
```bash
Host ai-sandbox
  HostName 127.0.0.1
  User vagrant
  Port 2222
  UserKnownHostsFile /dev/null
  StrictHostKeyChecking no
  PasswordAuthentication no
  # IMPORTANT: Replace 'ihr-name' with your actual Linux Mint username!
  IdentityFile /home/ihr-name/ai-sandbox/.vagrant/machines/default/virtualbox/private_key
  IdentitiesOnly yes
  LogLevel FATAL
```

Whenever you want to ssh into the VM:
```bash
ssh ai-sandbox
```

### 4. (optional) 
Connect IDE
- Open VS Code on your host.
- Press F1 -> Remote-SSH: Connect to Host...
- Select ai-sandbox.
- Once connected, open the /home/vagrant/workspace folder. This folder syncs directly with ~/ai-sandbox/workspace on your host!


## Daily Workflow & Maintenance

Because your workspace and Git SSH keys are stored on the host and mounted into the VM, you can safely destroy the VM if the AI agent breaks something.

### Stop the VM (Pause work):
```bash
    vagrant halt
```

### Start the VM (Resume work):
```bash
    vagrant up
```

### Nuke and Rebuild (The "Undo" button):
    If the environment gets corrupted, destroy it and recreate it. Your code and GitHub keys will survive!
```bash
    vagrant destroy -f
    vagrant up
```

## Important Notes for AI Agents

### Docker commands:
The VM is configured to alias docker to podman. When the AI agent tries to run docker run ..., it will seamlessly execute via rootless Podman.

### Ports:
Port 8080 is forwarded to your host. If the agent runs a Spring Boot app on port 8080 in the VM, you can view it on your host at http://localhost:8080.
